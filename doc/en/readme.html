Links Management

Author: Amir Soleimani
NVDA compatibility: 2026.1 and later

Overview

Links Management is an opt-in NVDA add-on that improves Browse Mode navigation when Screen Layout is turned off.

When enabled, each link behaves as a separate line for Up/Down arrow navigation. Trailing punctuation is intelligently attached to the link line to prevent blank or punctuation-only lines.

Inspired by JAWS, this add-on focuses on clarity, predictability, and navigation control.

Settings

A dedicated settings panel is available at:

NVDA menu → Preferences → Settings → Links management

Enable Links management

When enabled (and Screen Layout is off):

  • Each link is treated as a separate navigation "line" in Browse Mode.
  • Trailing punctuation is attached to the link line.
  • Blank or punctuation-only lines are avoided.

Notes

  • This add-on only affects Browse Mode behavior.
  • It is opt-in and must be enabled in Settings.
  • Designed for users who prefer structured link-by-link navigation.

Version 1.0.1 changes

  • Fixed an issue where certain interactive elements exposed as buttons (e.g. <a role="button"> on Google “Search instead for…” suggestions) could be merged into a single line when Screen layout was off.
  • The add-on now preserves NVDA’s original line boundaries and only applies link line-splitting when actual links exist within the current NVDA line.
  • Improved semantic handling to avoid interfering with elements that behave like buttons.

Version 1.0.2 changes

  • Added support for attaching the bullet character (•) to the preceding link line, preventing it from being placed on a separate line in Browse Mode when Screen layout is off.
  • Added support for keeping simple slash breadcrumb separators (/) on the same line as the preceding link.
  • This improves navigation in content where links are separated by bullets or breadcrumb separators.

Version 1.1 changes

  • Improved handling of list bullets before links. Simple bullet markers such as • are now kept on the same line as the following link, for example • FSCompanion.
  • Improved handling of common separators after links. Separators such as /, ·, and | are now kept on the same line as the preceding link instead of being placed on separate lines.
  • Improved handling of Wikipedia-style reference links. Split references such as [۱ followed by ] are now kept together as [۱]. Persian, Arabic, and Latin digits are supported.
  • Improved handling of short parenthesized links, such as degree abbreviations on Wikipedia pages. Cases like (BA) and (JD) should no longer be split with the opening parenthesis on a separate line.
  • Fixed an over-aggressive merge involving numbered table-of-contents markers. Numbered markers such as 1., 1.1, and similar entries are kept separate from the following link, matching NVDA with screen layout enabled and the behavior observed in JAWS.
  • Preserved previous safeguards against unwanted structural merging, including heading/paragraph separation, list boundary handling, and Down Arrow navigation traps.

Version 1.2.0 changes

  • Fixed cases where colons after links could appear on a separate line. For example, Synonyms of temblor: on Merriam-Webster is now kept on one line.
  • Improved handling of ordinary numbered lists. In cases such as the nvda.ru VK Feed list, numbers like 1. are now kept on the same line as the following link when this matches NVDA with Screen Layout enabled.
  • Improved handling of ordered list items that contain a link followed by a colon and description. For example, Sintra.ai shortlist items are now presented as 1. Sintra AI: followed by the description on the next line.
  • Improved Wikipedia reference handling. References such as [۱]. are now kept together instead of splitting the closing bracket and period onto a separate line.
  • Improved handling of parenthesized links, including multi-word link text, while preserving normal Up and Down Arrow navigation.
  • Fixed an issue where standalone right-to-left direction marks before links could appear on their own line. They are now kept with the following link, for example in expanded YouTube descriptions.
  • Preserved previous safeguards against unwanted structural merging and cursor traps, including Down Arrow and Up Arrow navigation through repaired lines.

Version 1.2.1

  • Fixed a critical cursor navigation issue on some Merriam-Webster pages where NVDA could get stuck on a link line followed by a colon-prefixed definition.
  • Improved handling of link lines followed by : description patterns. The colon is now kept with the preceding link, while the following definition or description remains on the next line.
  • Preserved previous safeguards against unwanted structural merging and Up/Down Arrow navigation traps.
